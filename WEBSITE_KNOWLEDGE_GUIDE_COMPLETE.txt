3. CẤU TRÚC DATABASE
================================================================================

BẢNG CHÍNH:

1. users
   - id (PK)
   - username (unique)
   - email (unique)
   - password (hashed)
   - user_role (0=user, 1=admin)
   - email_verified_at
   - customer_name
   - phone
   - avatar
   - created_at, updated_at

2. fruits (products)
   - id (PK)
   - name
   - description
   - price (integer, VND)
   - image
   - category_id (FK)
   - created_at, updated_at

3. categories
   - id (PK)
   - name
   - description
   - created_at, updated_at

4. carts
   - id (PK)
   - user_id (FK, nullable for guests)
   - session_id (for guest carts)
   - created_at, updated_at

5. cart_items
   - id (PK)
   - cart_id (FK)
   - fruit_id (FK)
   - qty
   - created_at, updated_at

6. orders
   - id (PK)
   - user_id (FK)
   - customer_name
   - total_amount
   - status (pending, processing, completed, cancelled)
   - created_at, updated_at

7. order_items
   - id (PK)
   - order_id (FK)
   - fruit_id (FK)
   - qty
   - price
   - created_at, updated_at

8. contact_notifications
   - id (PK)
   - name
   - email
   - message
   - created_at, updated_at

9. product_reviews
   - id (PK)
   - user_id (FK)
   - fruit_id (FK)
   - rating (1-5)
   - comment
   - created_at, updated_at

10. website_settings
    - id (PK)
    - setting_key (unique)
    - setting_value
    - created_at, updated_at

11. sliders
    - id (PK)
    - title
    - description
    - image
    - link
    - is_active
    - created_at, updated_at

4. BACKEND ARCHITECTURE (LARAVEL)
================================================================================

CẤU TRÚC THƯ MỤC:
app/
├── Http/
│   ├── Controllers/
│   │   ├── Controller.php (Base controller)
│   │   ├── HomeController.php (Frontend pages)
│   │   ├── ProductController.php (Product display)
│   │   ├── CartController.php (Cart operations)
│   │   ├── CheckoutController.php (Order processing)
│   │   ├── UserController.php (User management)
│   │   └── admin/ (Admin controllers)
│   ├── Middleware/
│   │   └── AdminMiddleware.php (Admin access control)
│   └── Requests/ (Form validation)
├── Models/ (Eloquent Models)
├── Providers/ (Service Providers)
└── Console/ (Commands)

MODELS QUAN TRỌNG:

1. User Model
   - Authentication & Authorization
   - Role-based access (user/admin)
   - Profile management

2. Product Model (Fruit)
   - Relationships: belongsTo(Category), hasMany(CartItem, OrderItem, Review)
   - Image handling
   - Price formatting

3. Cart & CartItem Models
   - Session-based for guests
   - User-based for logged-in users
   - Quantity management

4. Order & OrderItem Models
   - Order status tracking
   - Total calculation
   - Customer information

MIDDLEWARE:
- AdminMiddleware: Protects admin routes
- Auth middleware: Laravel built-in
- CSRF protection: Automatic

12. DEPLOYMENT GUIDE
================================================================================

SERVER REQUIREMENTS:
- PHP 8.1+
- MySQL 8.0+
- Composer
- Node.js & NPM
- Web server (Apache/Nginx)

DEPLOYMENT STEPS:
1. Upload files to server
2. Install dependencies: composer install
3. Setup environment: cp .env.example .env
4. Generate key: php artisan key:generate
5. Run migrations: php artisan migrate
6. Seed database: php artisan db:seed
7. Build assets: npm run build
8. Set permissions for storage/
9. Configure web server
10. Setup SSL certificate
